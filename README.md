# Threads Monitoring System (Python)

Этот скрипт позволяет мониторить новые посты в Threads по ключевым словам, используя Google Search (через SerpApi) для поиска ссылок и Apify для сбора детальных данных (лайки, текст).

## Возможности
- Поиск постов в Threads за последние 4 дня (`tbs=qdr:d4`).
- Фильтрация дубликатов с использованием базы данных SQLite.
- Скрейпинг данных (лайки, текст, дата) через Apify.
- Анализ ТОП-5 постов по количеству лайков.
- Сохранение результатов в CSV.

## Установка
1. Убедитесь, что у вас установлен Python.
2. Установите необходимые зависимости:
   ```bash
   pip install -r requirements.txt
   ```

## Настройка
1. Откройте `main.py` и вставьте ваши API-ключи:
   - `SERPAPI_KEY`: Получить на [serpapi.com](https://serpapi.com/)
   - `APIFY_TOKEN`: Получить на [apify.com](https://apify.com/)
2. (Опционально) Создайте файл `.env` на основе `.env.template` и укажите ключи там.

## Запуск
Запустите скрипт командой:
```bash
python main.py
```
Введите ключевое слово при запросе, и скрипт выполнит поиск и сбор данных.

## Файлы
- `main.py`: Основной код системы.
- `history.db`: Локальная база данных SQLite для хранения обработанных ссылок.
- `threads_monitoring_results.csv`: Файл с результатами мониторинга.
- `requirements.txt`: Список необходимых библиотек.
